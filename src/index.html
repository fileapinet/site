For example, {{ api_base }}/v1/virus-scan?key=XXXXXXXX

There are three ways to input a file:

  • A GET request with a URL to the file to test in the query string's `source` parameter, e.g. {{ api_base }}/v1/virus-scan?source=http://your-site.com/uploads/suspicious.doc
  • A POST request with a URL to the file to test in the request body's `source` parameter.
  • A POST request with the file's binary content in the request body, named `source`.

If you have different environments (e.g. dev, staging and production), or different teams or clients that should be billed internally for their usage, you can add a 'billingGroup' parameter to the API URL's query string, for example, '{{ api_base }}/v1/virus-scan?billingGroup=clientA-production'. We can then send you a receipt of usage grouped by this key.

Client libraries

CLI (Bash)

To install the command line library, download it to your system, and make it executable:

sudo wget fileapi.net/cli -O /usr/bin/fileapi
sudo chmod a+x /usr/bin/fileapi

You need to set your API key in an environment variable. For example, export FILEAPI_KEY=your-api-key.

Now you can run, for example, fileapi convert-image-to-other-formats http://demo-api.fileapi.dev/fixtures/brain.jpg

For help, run fileapi -h
